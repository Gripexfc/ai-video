## 一、项目简介

视频miao~（视频miao~）是一个 **全本地运行的 AI 短剧/漫画生成工具**。从剧本编写到最终视频合成，所有步骤可在本地完成，数据不上传。

### 核心创作流程

```
输入故事梗概 → AI 生成多集剧本 → AI 提取角色/场景/道具
                                    ↓
                           AI 生成分镜脚本
                                    ↓
              AI 生成角色图 / AI 生成场景图 / AI 生成道具图
                                    ↓
                         AI 生成分镜静帧图
                                    ↓
                        AI 生成分镜视频片段
                                    ↓
                      自动合并为完整集视频
```

### 系统架构

| 组件 | 技术栈 | 端口 |
|------|--------|------|
| 后端 | Node.js + Express + SQLite | 5679 |
| 前端 | Vue 3 + Vite + Element Plus | 3013 |
| 桌面壳 | Electron 28 | — |

## 二、安装与启动

### 2.1 桌面版（Windows exe）

1. 前往 GitHub **Releases** 页面下载：
   - `视频miao~ Setup x.x.x.exe` — NSIS 安装包（推荐）
   - `视频miao~ x.x.x.exe` — 便携版，双击即用
2. 首次启动后，配置文件自动生成在：
   ```
   %APPDATA%\视频miao~\backend\configs\config.yaml
   ```
3. 点击右上角 **「AI 配置」**，填入 API Key 即可使用。

### 2.2 开发模式（macOS / Linux / Windows）

**环境要求：** Node.js >= 18

```bash
# 1. 克隆项目
git clone <repo-url>
cd 视频miao~

# 2. 启动后端
cd backend-node
npm install
cp configs/config.example.yaml configs/config.yaml   # 复制配置模板
npm run migrate                                        # 初始化数据库
npm start                                              # 启动，端口 5679

# 3. 新开终端，启动前端
cd frontweb
npm install
npm run dev                                            # 端口 3013，自动代理到后端
```

浏览器访问 `http://localhost:3013`。

**一键启动（Windows）：** 双击 `run_dev.bat` 或 `run_dev.ps1`。

## 四、AI 配置详解（重点）

### 4.1 配置入口与结构

**入口：** 点击软件右上角 **「AI 配置」** 按钮。

AI 配置存储在 **SQLite 数据库**（`ai_service_configs` 表）中，通过 Web UI 管理，无需手动编辑文件。

配置页面分为 **多个 Tab**，每个 Tab 对应一类 AI 服务：

```
┌──────────────────────────────────────────────────┐
│  [文本生成]  [图片生成]  [分镜图片]  [视频生成]  [TTS 语音] │
├──────────────────────────────────────────────────┤
│                                                  │
│  配置卡片 1：名称 / 服务商 / 模型 / [测试] [编辑] [删除]  │
│  配置卡片 2：...                                        │
│  ...                                                   │
│                                                  │
│  [+ 新增配置]  [一键配置通义]  [一键配置火山]  [导入配置]     │
└──────────────────────────────────────────────────┘
```

每类服务可配置 **多条记录**（不同服务商或不同模型），通过 `is_default` 标记默认使用的配置。

### 4.2 五大服务类型说明

| 服务类型 (service_type) | 用途 | 必需性 |
|---|---|---|
| **`text`** 文本生成 | 生成剧本、提取角色/场景、生成分镜脚本、提示词优化 | 必配 |
| **`image`** 图片生成 | 生成角色图、场景图、道具图（**无参考图**，纯文生图） | 必配 |
| **`storyboard_image`** 分镜图片 | 生成分镜静帧图（**支持参考图**：角色+场景叠加） | 必配 |
| **`video`** 视频生成 | 将分镜静帧转为视频片段 | 必配 |
| **`tts`** 语音合成 | 为分镜添加旁白配音 | 可选 |

> `image` 与 `storyboard_image` 的区别：`image` 用于角色/场景/道具的「初始形象生成」（纯文字描述 → 图片）；`storyboard_image` 用于「分镜画面生成」（文字 + 角色参考图 + 场景参考图 → 合成画面）。

### 4.3 手动添加 AI 配置（逐字段说明）

点击 **「+ 新增配置」** 按钮，弹出配置表单：

| 字段 | 说明 | 示例 |
|------|------|------|
| **配置名称** (name) | 自定义名称，方便识别 | "通义文本" / "豆包视频" |
| **服务商** (provider) | 服务商标识名，自由填写 | `openai` / `dashscope` / `volcengine` |
| **接口规范** (api_protocol) | **关键字段**，决定 API 调用格式（详见第六节对照表） | `openai` / `volcengine` / `dashscope` / `gemini` |
| **Base URL** (base_url) | API 基础地址 | `https://dashscope.aliyuncs.com/compatible-mode/v1` |
| **API Key** (api_key) | 你的密钥 | `sk-xxxxxxxx` |
| **模型** (model) | 可用模型列表（下拉选择） | `qwen-plus` / `doubao-seedance-1-0-pro-250528` |
| **默认模型** (default_model) | 实际使用的模型 | 同上 |
| **API 端点** (endpoint) | 自定义创建端点（可选，留空用默认） | `/v1/images/generations` |
| **查询端点** (query_endpoint) | 自定义查询端点（异步任务用） | `/volc/v1/contents/generations/tasks/{task_id}` |
| **设为默认** (is_default) | 是否作为该类型的默认配置 | ✅ |
| **启用** (is_active) | 是否启用此配置 | ✅ |

> **接口规范 (api_protocol) 是最重要的字段。** 它告诉系统用哪种协议格式来调用 API。选错会导致请求失败。如果留空，系统会根据服务商名自动推断。

### 4.4 一键配置模板

页面顶部提供两个快捷按钮：

- **「一键配置通义」** → 自动创建阿里云 DashScope 的 text + image + storyboard_image + video 四套配置
- **「一键配置火山」** → 自动创建火山引擎 Volcengine 的 text + image + storyboard_image + video 四套配置

使用方式：点击一键配置 → 每条配置填入你的 API Key → 保存。

### 4.5 导入中转站预置配置

项目自带多个中转站 JSON 配置文件（位于 `各大平台中转站配置/` 目录），涵盖国内主流 AI API 中转服务：

| 文件 | 中转站 | Base URL |
|------|--------|----------|
| `302ai-302.json` | 302.AI | `https://api.302.ai` |
| `geeknow.json` | GeekNow | — |
| `n1n.json` | N1N | — |
| `向量.json` | 向量 | — |
| `云雾ai.json` | 云雾 AI | — |
| `飞儿api-ffir.cn.json` | 飞儿 API | `https://ffir.cn` |

**导入步骤：**
1. 点击 **「导入配置」** 按钮
2. 选择一个 JSON 文件
3. 系统自动批量创建所有配置条目
4. **重要：** 导入后，点击每条配置的「编辑」，将 `api_key` 从 `sk-1234` 改为你自己的真实 API Key

> 中转站的好处：一个 API Key 即可调用多种模型（DeepSeek、豆包、Gemini、可灵、Vidu 等），无需分别在各平台注册。

### 4.6 连接测试

每条配置右侧有 **「测试」** 按钮，点击后系统会发送一条简短请求验证连通性：
- ✅ 绿色提示 = 连接成功
- ❌ 红色提示 = 具体错误信息（认证失败/模型不存在/网络不通等）

**建议：** 每次新增或修改配置后，先点测试确认通过再使用。

## 五、各 AI 服务商配置教程

### 5.1 阿里云 DashScope（通义千问）

**申请 API Key：**
1. 访问 [阿里云百炼控制台](https://bailian.console.aliyun.com/)
2. 注册/登录阿里云账号
3. 进入「模型广场」，开通需要的模型
4. 左侧菜单 →「API-KEY 管理」→ 创建 API Key（`sk-` 开头）
5. 新用户通常有免费额度

**配置参数：**

| 项目 | 值 |
|------|---|
| 服务商 | `dashscope` |
| 接口规范 | `dashscope` |
| Base URL | `https://dashscope.aliyuncs.com/compatible-mode/v1` |

**文本模型推荐：**

| 模型 | 特点 |
|------|------|
| `qwen-turbo` | 速度快、成本低，适合批量 |
| `qwen-plus` | 均衡，推荐日常 |
| `qwen-max` | 最强文本，推荐剧本生成 |
| `qwen-long` | 超长上下文，适合长剧本 |

**图片模型推荐：**

| 模型 | 特点 |
|------|------|
| `wanx2.1-t2i-turbo` | 速度快，通用 |
| `wanx2.1-t2i-plus` | 更高质量 |

**视频模型推荐：**

| 模型 | 特点 |
|------|------|
| `wan2.1-t2v-turbo` | 文字转视频，快 |
| `wan2.1-t2v-plus` | 更高质量 |

**文本模型：**

|------|---------|

| `Doubao-pro-128k` | `doubao-pro-128k-241215` |

**图片模型：**

| 模型 | 端点 ID |
|------|---------|

| `Doubao-seedream-5.0` | `doubao-seedream-5-0-260128` |

**视频模型：**

| 模型 | 端点 ID | 说明 |

## 六、接口规范（api_protocol）完整对照表

接口规范决定了系统发送 API 请求时的**数据格式和调用方式**。选错会导致请求失败。

### 图片类接口规范

| api_protocol | 适用服务商 | 说明 |
|---|---|---|
| `openai` | 通用 | 标准 `/images/generations` 接口 |
| `dashscope` | 阿里云 | DashScope 万象图片 API |
| `volcengine` | 火山引擎 | 豆包 Seedream 图片 API |
| `gemini` | Google | Gemini `generateContent` 图片生成 |
| `nano_banana` | NanoBanana | NanoBanana 图片生成（异步轮询） |
| `kling` | 可灵 | 可灵 AI 图片生成（异步轮询） |

### 视频类接口规范

| api_protocol | 适用服务商 | 说明 |
|---|---|---|
| `openai` | 通用 | 标准 OpenAI 视频接口 |
| `volcengine` | 火山引擎 | Seedance 1.0/1.5 经典单链路 |
| `volcengine_omni` | 火山引擎 | **Seedance 2.0 多参考图模式**（配合分镜全能模式） |
| `dashscope` | 阿里云 | Wan 视频 |
| `gemini` | Google | Gemini Veo |
| `sora` | OpenAI | Sora（通过代理，multipart/form-data） |
| `veo3` | Google | Veo3（JSON + images 数组） |
| `vidu` | Vidu | Vidu 官方/代理 |
| `kling_omni` | 可灵 | 可灵 Omni 视频（JWT 或 Bearer） |
| `xai` | xAI/Grok | Grok Imagine 视频 |
| `jimeng_ai_api` | 即梦 | 自建即梦 Seedance 代理 |

### 自动推断规则

当 `api_protocol` 留空时，系统根据 provider 名推断：

| provider 包含 | 推断为 |
|---|---|
| `dashscope` / `qwen_image` | `dashscope` |
| `volces` / `volcengine` / `volc` | `volcengine` |
| `gemini` / `google` | `gemini` |
| `nano_banana` | `nano_banana` |
| `kling` / `klingai` | `kling` |
| `vidu` | `vidu` |
| `xai` / `grok` | `xai` |
| 其他 | `openai` |

## 七、中转站配置 JSON 格式说明

中转站 JSON 文件是一个数组，每个元素对应一条 AI 配置。导入后系统会批量创建。

```json
[
  {
    "service_type": "text",
    "provider": "自定义",
    "api_protocol": "",
    "name": "配置显示名",
    "base_url": "https://api.example.com/v1",
    "api_key": "sk-你的key",
    "model": ["model-a", "model-b"],
    "default_model": "model-a",
    "endpoint": "",
    "query_endpoint": "",
    "priority": 0,
    "is_default": true,
    "is_active": true,
    "settings": null
  }
]
```

**字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `service_type` | string | `text` / `image` / `storyboard_image` / `video` / `tts` / `stt` |
| `provider` | string | 服务商标识，自由填写 |
| `api_protocol` | string | 接口规范，留空自动推断 |
| `name` | string | 配置名称，显示在 UI 卡片上 |
| `base_url` | string | API 基础地址 |
| `api_key` | string | **导入后务必改成你自己的 Key** |
| `model` | string[] | 可选模型列表（下拉菜单） |
| `default_model` | string | 默认使用的模型 |
| `endpoint` | string | 自定义创建端点，留空用协议默认 |
| `query_endpoint` | string | 异步任务的查询端点，支持 `{task_id}` 占位符 |
| `priority` | number | 优先级（保留字段） |
| `is_default` | boolean | 是否为该类型的默认配置 |
| `is_active` | boolean | 是否启用 |
| `settings` | object\|null | 扩展设置（如 TTS 的 voice_id） |

## 八、完整创作流程

### 第一步：配置 AI 服务

进入 **「AI 配置」** 页面，至少配置以下四类：
1. **文本生成** — 用于剧本/分镜/提示词（如 `qwen-plus` 或 `deepseek-v3.2`）
2. **图片生成** — 用于角色/场景/道具图（如 `doubao-seedream-4.5` 或 `wanx2.1-t2i-turbo`）
3. **分镜图片** — 用于分镜画面（如 `doubao-seedream-4.5` 或 `gemini-2.5-flash-image`）
4. **视频生成** — 用于分镜视频（如 `doubao-seedance-1.0-pro-fast`）

每条配置填完后点 **「测试」** 确认连通。

### 第二步：创建新项目

1. 首页点击 **「+ 新建」**
2. 填写项目名称
3. 进入制作页

### 第三步：生成剧本

1. 在制作页输入 **故事梗概** 和 **风格设定**
2. 点击 **「生成剧本」** → AI 自动生成多集剧本
3. 可手动编辑每一集的文本

### 第四步：生成角色/场景/道具

1. 点击 **「提取角色」** → AI 自动从剧本提取角色列表
2. 点击 **「提取场景」** → AI 自动提取场景
3. 点击 **「提取道具」** → AI 自动提取道具
4. 对每个角色/场景/道具点击 **「生成图片」**
5. 检查生成结果，不满意可重新生成

### 第五步：生成分镜

1. 选择一集，点击 **「生成分镜」** → AI 自动创建分镜列表
2. 每条分镜包含：景别、运镜、对白、动作描述
3. 可手动编辑每条分镜

### 第六步：生成分镜图片与视频

1. 点击 **「生成分镜图片」** → AI 为每条分镜生成静帧画面
2. 检查效果后，点击 **「生成分镜视频」** → AI 将静帧转为视频片段
3. 系统默认 3 个并发任务，自动重试（最多 3 次）

### 第七步：合成视频

1. 所有分镜视频生成完毕后，点击 **「合成视频」**
2. 系统自动用 ffmpeg 将所有片段合并为完整的一集视频
3. 下载成品视频

### 一键流水线

制作页提供 **「一键流水线」** 按钮，自动按顺序执行上述所有步骤，无需手动逐步操作。

## 九、高级功能

### 9.1 分镜全能模式（v1.2.5+）

单个分镜可切换为「全能模式」：
- 编辑区输入**片段描述**
- 用 `@图片1`、`@图片2`... 语法引用参考图
- 参考图顺序一般为：场景 → 角色 → 道具 → 分镜主图
- `@图片N` 后加**半角空格**
- 此模式下生视频只发送这段文本，不拼接结构化提示词

**视频配置需选对应协议：**
- Seedance 2.0 → `volcengine_omni`
- 可灵 Omni → `kling_omni`

### 9.2 模型映射（ai_model_map）

可通过数据库 `ai_model_map` 表为不同任务场景指定不同的 AI 配置。例如：
- 剧本生成用 Qwen-Max
- 提示词优化用 DeepSeek
- 角色提取用 Qwen-Turbo

### 9.3 提示词覆写（prompt_overrides）

在设置中可自定义各类提示词模板，覆盖系统默认的 prompt。用于微调生成风格和效果。

### 9.4 素材库复用

角色库、场景库、道具库跨项目通用。在一个项目中生成的素材可导入其他项目使用。

### 9.5 项目导入导出

- **导出：** 首页点击项目卡片上的「导出」→ 下载 ZIP 文件
- **导入：** 首页点击「导入」→ 选择 ZIP 文件

### 9.6 风格预设

系统内置多种风格预设（动漫、写实、水彩等），可在制作页的风格选择器中切换。

## 十二、常见问题 FAQ

### Q: API Key 填错了怎么办？
在 AI 配置页面点击对应配置的「编辑」，修改 Key 后保存即生效。

### Q: 图片生成报错 "image size must be at least 3686400 pixels"？
火山引擎图片 API 的最低像素要求。系统会自动计算合适分辨率，通常无需处理。如仍报错检查是否自定义了 size 参数。

### Q: 视频生成报错 "model does not exist"？
确认已在火山方舟控制台开通该模型，使用正确的端点 ID。系统内置名称映射，两种写法（展示名/端点ID）均可。

### Q: 生成速度慢？
- 图片：15-60 秒
- 视频：1-5 分钟
- 建议用带 `turbo` / `fast` 后缀的模型
- 429 限流系统自动重试

### Q: 后端启动报错 "Cannot find module 'better-sqlite3'"？
```bash
cd backend-node && npm install
```
确认 Node.js >= 18。

### Q: 前端报错 "Failed to fetch"？
确认后端已启动（端口 5679），检查 `frontweb/vite.config.js` 中 proxy 指向。

### Q: 如何用中转站？
1. 去 `各大平台中转站配置/` 目录选一个 JSON 文件
2. 在 AI 配置页点「导入配置」→ 选文件
3. 每条配置点编辑 → 把 `api_key` 改成你从中转站获得的 Key
4. 点「测试」确认

### Q: 支持 Mac/Linux 吗？
后端（Node.js）和前端（Vue 3）理论上跨平台，但桌面版打包仅配置了 Windows。开发模式可在 Mac/Linux 运行。

### Q: insecure_tls 是什么？
设为 `true` 时跳过 HTTPS 证书验证。使用自签名证书的中转站时需要开启。
